---
name: UI integration style
description: User values seamless, well-integrated UI — not bolted-on features
type: feedback
---

User explicitly praised "seamless, great UI integration" after the comments feature was built as a slide-in drawer with sidebar preview cards rather than a separate page or modal.

**Why:** User appreciates when new features feel native to the existing design system (same card styles, fonts, color tokens) rather than visually disconnected.

**How to apply:** When adding new features, match the existing Away Design System tokens exactly (Midnight/Dusk/Sand palette, DM Sans + Georgia + JetBrains Mono hierarchy). Prefer contextual patterns (drawers, inline sections, cards in existing sidebar) over new pages or generic modals.
